# 材料管理系统 - 离线部署指南（不联网国产电脑）

## 🎯 适用场景

- 单位国产电脑不连接互联网
- 无法使用 `pip` 在线安装依赖
- 无法下载 `PyInstaller` 等工具
- 需要批量部署到多台内网电脑

---

## 📦 方案一：使用UOS系统自带Python（推荐）

### 原理
统信UOS系统自带Python3和大部分依赖库，只需要复制项目源码即可运行！

### 步骤1：在联网电脑准备依赖包

```bash
# 在联网的UOS电脑上执行

# 1. 创建依赖包下载目录
mkdir -p ~/offline_packages
cd ~/offline_packages

# 2. 下载所有依赖包（使用清华镜像加速）
pip download PyQt5==5.15.9 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download python-docx==0.8.11 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download openpyxl==3.1.0 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download PyMuPDF==1.21.0 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download jieba==0.42.1 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download wordcloud==1.8.1 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download Pillow==9.5.0 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip download chardet==5.1.0 -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 查看下载的文件
ls -lh ~/offline_packages/
```

### 步骤2：准备部署包

```bash
# 创建部署包目录
mkdir -p ~/材料管理系统部署包
cd ~/材料管理系统部署包

# 1. 复制项目源码
cp -r /path/to/Document\ Management\ System ./source

# 2. 复制依赖包
cp -r ~/offline_packages ./packages

# 3. 创建安装脚本
cat > install.sh << 'EOF'
#!/bin/bash
# 材料管理系统 - 离线安装脚本

echo "========================================"
echo "  材料管理系统 - 离线安装"
echo "========================================"
echo ""

# 检查是否在source目录
if [ ! -f "source/main.py" ]; then
    echo "[错误] 找不到 source/main.py"
    echo "请确保在部署包目录中运行此脚本"
    exit 1
fi

# 安装系统依赖（UOS自带，一般不需要）
echo "[信息] 检查系统依赖..."
sudo apt update
sudo apt install -y python3 python3-pip fonts-wqy-zenhei fonts-wqy-microhei

# 创建虚拟环境
echo "[信息] 创建虚拟环境..."
cd source
python3 -m venv venv
source venv/bin/activate

# 离线安装依赖
echo "[信息] 安装Python依赖（离线）..."
pip install --no-index --find-links=../packages -r requirements.txt

# 设置执行权限
echo "[信息] 设置权限..."
chmod +x main.py

# 创建启动脚本
cat > start.sh << 'STARTEOF'
#!/bin/bash
cd "$(dirname "$0")"
source venv/bin/activate
python main.py
STARTEOF
chmod +x start.sh

echo ""
echo "========================================"
echo "  安装完成！"
echo "========================================"
echo ""
echo "运行方式："
echo "  1. 双击运行：./start.sh"
echo "  2. 或执行：source venv/bin/activate && python main.py"
echo ""
EOF

chmod +x install.sh

# 4. 创建说明文件
cat > README.txt << 'EOF'
材料管理系统 - 离线部署包
========================

目录结构：
├── source/          # 项目源代码
├── packages/        # Python依赖包（.whl文件）
├── install.sh       # 安装脚本
└── README.txt       # 本说明文件

安装步骤：
1. 将整个文件夹复制到目标UOS电脑
2. 打开终端，进入此文件夹
3. 执行：./install.sh
4. 安装完成后，进入source目录运行：./start.sh

注意事项：
- 需要管理员权限（sudo）
- 安装过程约需5-10分钟
- 数据会自动保存在用户目录：~/.document_manager/
EOF

echo "部署包已创建在：~/材料管理系统部署包/"
```

### 步骤3：复制到内网电脑并安装

```bash
# 在目标UOS电脑上（不联网）

# 1. 插入U盘，复制部署包到电脑
# 2. 打开终端，进入部署包目录
cd /media/用户名/U盘/材料管理系统部署包

# 3. 执行安装
./install.sh

# 4. 运行程序
cd source
./start.sh
```

## 📦 方案二：利用UOS自带Python库（极简方案）

### 原理
UOS系统自带的Python3已经包含了很多常用库，我们只需要安装缺失的即可。

### 检查UOS自带库

```bash
# 在UOS上执行，查看已有库
python3 -c "import PyQt5; print('PyQt5已安装')"
python3 -c "import sqlite3; print('sqlite3已安装')"
python3 -c "import PIL; print('Pillow已安装')"
```

### 创建最小依赖包

```bash
# 只下载UOS没有的依赖
mkdir ~/minimal_packages
cd ~/minimal_packages

# 这些通常是UOS没有的
pip download python-docx==0.8.11
pip download openpyxl==3.1.0
pip download PyMuPDF==1.21.0
pip download jieba==0.42.1
pip download wordcloud==1.8.1
pip download chardet==5.1.0
```

## 📦 方案三：直接运行源码（无需安装依赖）

### 前提条件
如果UOS系统已经安装了以下软件：
- Python3
- python3-pyqt5 (apt安装)
- python3-pil (apt安装)

### 修改代码适配系统库

```bash
# 修改requirements.txt，移除系统已提供的库
cat > requirements_minimal.txt << 'EOF'
# 仅包含UOS apt没有的包
python-docx==0.8.11
openpyxl==3.1.0
PyMuPDF==1.21.0
jieba==0.42.1
wordcloud==1.8.1
chardet==5.1.0
EOF
```

### 使用系统Python直接运行

```bash
# 不需要虚拟环境，直接用系统Python

# 1. 安装系统依赖
sudo apt install -y python3-pyqt5 python3-pil fonts-wqy-zenhei

# 2. 安装缺失的Python包（离线）
pip3 install --user --no-index --find-links=./packages -r requirements_minimal.txt

# 3. 直接运行
python3 main.py
```

## 📦 方案四：Docker方式（高级用户）

### 在有Docker的联网电脑打包

```dockerfile
# Dockerfile
FROM python:3.9-slim

WORKDIR /app

# 复制项目文件
COPY . /app

# 安装依赖
RUN pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 运行程序
CMD ["python", "main.py"]
```

```bash
# 构建镜像
docker build -t document-manager .

# 导出镜像
docker save document-manager > document-manager.tar
```

### 在内网电脑导入运行

```bash
# 安装Docker（UOS软件商店有）
sudo apt install docker.io

# 导入镜像
docker load < document-manager.tar

# 运行（需要X11转发）
docker run -e DISPLAY=$DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix document-manager
```

## 🔧 快速部署脚本（一键安装）

创建 `快速部署.sh`：

```bash
#!/bin/bash
# 材料管理系统 - 快速离线部署脚本

set -e

echo "========================================"
echo "  材料管理系统 - 快速部署"
echo "========================================"
echo ""

# 配置
INSTALL_DIR="$HOME/材料管理系统"
PACKAGES_DIR="$(dirname "$0")/packages"
SOURCE_DIR="$(dirname "$0")/source"

# 检查目录
if [ ! -d "$PACKAGES_DIR" ]; then
    echo "[错误] 找不到packages目录"
    exit 1
fi

if [ ! -d "$SOURCE_DIR" ]; then
    echo "[错误] 找不到source目录"
    exit 1
fi

echo "[1/5] 安装系统依赖..."
sudo apt update
sudo apt install -y python3 python3-venv python3-pip fonts-wqy-zenhei fonts-wqy-microhei

echo "[2/5] 创建安装目录..."
mkdir -p "$INSTALL_DIR"
cp -r "$SOURCE_DIR"/* "$INSTALL_DIR/"

echo "[3/5] 创建虚拟环境..."
cd "$INSTALL_DIR"
python3 -m venv venv
source venv/bin/activate

echo "[4/5] 安装Python依赖（离线）..."
pip install --no-index --find-links="$PACKAGES_DIR" -r requirements.txt

echo "[5/5] 创建启动器..."
cat > "$HOME/材料管理系统.sh" << EOF
#!/bin/bash
cd "$INSTALL_DIR"
source venv/bin/activate
python main.py
EOF
chmod +x "$HOME/材料管理系统.sh"

# 创建桌面快捷方式
cat > "$HOME/Desktop/材料管理系统.desktop" << EOF
[Desktop Entry]
Name=材料管理系统
Comment=党政机关材料管理软件
Exec=$HOME/材料管理系统.sh
Icon=$INSTALL_DIR/icon.png
Type=Application
Terminal=false
Categories=Office;
EOF
chmod +x "$HOME/Desktop/材料管理系统.desktop"

echo ""
echo "========================================"
echo "  部署完成！"
echo "========================================"
echo ""
echo "启动方式："
echo "  1. 双击桌面图标"
echo "  2. 或运行：$HOME/材料管理系统.sh"
echo ""
echo "数据目录：$HOME/.document_manager/"
echo ""
```

## 🎯 推荐方案总结

| 方案 | 复杂度 | 依赖数量 | 适用场景 |
|------|--------|----------|----------|
| 方案一：完整离线包 | 中 | 全部 | 通用，最稳定 |
| 方案二：最小依赖包 | 低 | 少量 | UOS已安装PyQt5 |
| 方案三：系统Python | 低 | 极少 | UOS完整安装 |
| 方案四：Docker | 高 | 容器内 | 有Docker环境 |

**推荐**：方案一（完整离线包），一次准备，到处部署！

## 📞 常见问题

### Q1：如何查看UOS安装了哪些Python包？

```bash
# 查看所有已安装的包
pip3 list

# 查看特定包
pip3 show PyQt5
```

### Q2：pip install --no-index 报错怎么办？

```bash
# 检查包是否完整
ls -lh packages/

# 尝试单独安装某个包
pip install --no-index --find-links=./packages PyQt5-*.whl

# 查看详细错误
pip install --no-index --find-links=./packages -r requirements.txt -v
```

### Q3：如何减小部署包体积？

```bash
# 只保留必要的.whl文件，删除以下文件：
# - *.tar.gz
# - *-doc-*.whl
# - *-tests-*.whl
# - *-examples-*.whl

# 压缩部署包
tar czvf 材料管理系统部署包.tar.gz 材料管理系统部署包/
```

### Q4：如何批量部署到多台电脑？

```bash
# 方法1：使用U盘复制
# 方法2：使用内网文件服务器
# 方法3：使用PXE网络启动

# 创建自动化部署脚本
cat > deploy_all.sh << 'EOF'
#!/bin/bash
# 批量部署脚本
for ip in 192.168.1.{101..110}; do
    echo "部署到 $ip..."
    scp -r 材料管理系统部署包 user@$ip:~/
    ssh user@$ip "cd ~/材料管理系统部署包 && ./install.sh"
done
EOF
```
