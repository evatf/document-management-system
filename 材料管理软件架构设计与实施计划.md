# 材料管理软件 - 架构设计与实施计划

## 一、项目概述

### 1.1 目标
开发一款面向党政机关的材料管理软件，支持文档归档、检索、智能分析等功能，适配统信UOS系统（Python 3.7.3）。

### 1.2 核心约束
- **操作系统**：统信UOS（国产Linux发行版）
- **Python版本**：3.7.3（需确保所有依赖兼容）
- **架构要求**：单体应用，避免前后端分离
- **兼容性**：优先选择成熟稳定的库，避免新特性

---

## 二、技术架构选型

### 2.1 GUI框架选择：PyQt5

**选择理由**：
- PyQt5在Python 3.7上非常成熟稳定
- 原生界面，无需Web服务器
- 丰富的控件库，支持文档预览
- 跨平台兼容性好
- 统信UOS对Qt框架支持良好

**版本锁定**：`PyQt5==5.15.9`（最后一个稳定支持Python 3.7的版本）

### 2.2 数据存储

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 元数据库 | SQLite | 单文件数据库，无需配置，Python内置支持 |
| 文档存储 | 文件系统 | 保持原始格式，数据库只存路径和元数据 |
| 配置文件 | JSON/YAML | 存储用户偏好、分类规则等 |

### 2.3 文档处理

| 功能 | 库 | Python 3.7兼容版本 |
|------|-----|-------------------|
| Word文档(.docx) | python-docx | 0.8.11 |
| PDF读取 | PyMuPDF / pdfplumber | PyMuPDF 1.21.0 |
| Excel(.xlsx) | openpyxl | 3.1.0 |
| 纯文本 | 内置 | - |
| 文件类型检测 | python-magic / puremagic | puremagic 1.14 |

### 2.4 NLP与智能分析

| 功能 | 库 | 版本 | 说明 |
|------|-----|------|------|
| 中文分词 | jieba | 0.42.1 | 轻量级，兼容性好 |
| 词性标注 | jieba | 内置 | - |
| 关键词提取 | jieba.analyse | 内置 | TF-IDF算法 |
| 文本相似度 | difflib + jieba | 内置+jieba | 余弦相似度 |
| 词云生成 | wordcloud | 1.8.1 | 需检查依赖 |
| 句法分析 | snownlp | 0.12.3 | 轻量NLP库 |

**注意**：避免使用spaCy、transformers等重型库，它们对Python 3.7支持有限且依赖复杂。

### 2.5 文档预览方案

1. **纯文本预览**：直接在QTextEdit中显示
2. **富文本预览**：转换为HTML后用QTextBrowser显示
3. **PDF预览**：
   - 方案A：使用PyMuPDF渲染为图片，在QLabel中显示
   - 方案B：调用系统默认PDF阅读器
4. **Word预览**：提取文本内容显示，或调用LibreOffice/WPS

### 2.6 文档比对方案

使用`difflib`库实现：
- `difflib.SequenceMatcher`：计算相似度
- `difflib.unified_diff`：生成差异
- 自定义双栏对比视图，高亮增删改

## 三、功能模块设计

### 3.1 材料库模块

#### 功能清单
- 文档导入（单文件/文件夹批量导入）
- 自动归档（基于规则识别文档类型和年份）
- 分类管理（层级分类，支持自定义）
- 标签管理（多标签，支持颜色）
- 文档元数据管理（标题、作者、日期、来源等）
- 文档预览
- 内容复制粘贴

#### 数据结构设计

```sql
-- 文档表
CREATE TABLE documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    file_path TEXT NOT NULL,
    file_type TEXT,
    file_size INTEGER,
    content_text TEXT,
    create_date DATE,
    import_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    category_id INTEGER,
    year INTEGER,
    author TEXT,
    source TEXT,
    notes TEXT,
    is_starred INTEGER DEFAULT 0,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);

-- 分类表
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    parent_id INTEGER,
    type TEXT DEFAULT 'custom',
    sort_order INTEGER DEFAULT 0,
    FOREIGN KEY (parent_id) REFERENCES categories(id)
);

-- 标签表
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    color TEXT DEFAULT '#3498db'
);

-- 文档-标签关联表
CREATE TABLE document_tags (
    document_id INTEGER,
    tag_id INTEGER,
    PRIMARY KEY (document_id, tag_id),
    FOREIGN KEY (document_id) REFERENCES documents(id),
    FOREIGN KEY (tag_id) REFERENCES tags(id)
);
```

#### 自动归档规则
1. **年份识别**：从文件名、文档内容、元数据中提取年份
2. **文档类型识别**：
   - 基于文件名关键词（如"通知"、"决定"、"报告"）
   - 基于文档内容特征
   - 用户自定义规则

### 3.2 文档检索模块

#### 检索方式
1. **全文检索**：搜索文档标题和内容
2. **条件筛选**：按分类、标签、年份、作者等
3. **组合检索**：多条件组合
4. **标签检索**：点击标签快速筛选

#### 实现方案
- 使用SQLite FTS5全文搜索扩展
- 或使用LIKE + Python内存过滤（兼容性更好）
- 检索结果高亮：自定义QSyntaxHighlighter

### 3.3 智能分析模块

#### 3.3.1 高频词统计
```
输入：文档/文件夹
处理：jieba分词 → 去停用词 → 词频统计 → 排序
输出：词云图片 / 词频列表（可导出）
```

#### 3.3.2 文稿比对
```
输入：文档A、文档B
处理：文本提取 → difflib比对 → 差异标记
输出：双栏对比视图（红色删除、绿色新增、黄色修改）
```

#### 3.3.3 常用句式提取
```
输入：文档
处理：分句 → 句法模式识别 → 频次统计
输出：句式列表（可收藏到金句库）
```
识别模式：
- 排比句（相似结构重复出现）
- 固定搭配（如"坚持...原则"、"以...为..."）
- 常用开头/结尾句式

#### 3.3.4 领导风格学习
```
输入：原始稿 + 领导修改稿（配对）
处理：差异分析 → 词汇偏好统计 → 句式偏好统计
输出：风格报告（新增词汇、删除词汇、偏好句式）
```

#### 3.3.5 相似度检测
```
输入：两篇文档
处理：分词 → 向量化 → 余弦相似度计算
输出：相似度百分比 + 相似段落对照
```

#### 3.3.6 结构分析
```
输入：文档
处理：段落识别 → 标题提取 → 关键句识别
输出：文档大纲树 + 段落结构图 + 关键句列表
```

### 3.4 星标收藏模块

```sql
-- 收藏表
```

功能：

- 收藏笔记

```sql

    id INTEGER PRIMARY KEY AUTOINCREMENT,

```

功能：
- 手动添加金句
- 从文档中摘录
- 分类管理

## 七、关键技术实现要点

### 7.1 统信UOS兼容性注意事项

1. **字体问题**：使用系统默认字体，避免指定特定字体
2. **文件路径**：使用`pathlib.Path`处理路径，兼容Linux
3. **文件编码**：统一使用UTF-8，处理中文时注意编码
4. **依赖安装**：优先使用pip wheel包，避免编译
5. **权限问题**：注意文件读写权限

### 7.2 大文件处理策略

1. **异步加载**：使用QThread处理大文件解析
2. **分页显示**：长文档分页加载
3. **缓存机制**：缓存解析结果
4. **进度提示**：显示处理进度条

### 7.3 搜索性能优化

1. **索引优化**：对常用搜索字段建立索引
2. **延迟搜索**：输入停止后500ms再搜索
3. **结果分页**：分批加载搜索结果
4. **缓存结果**：缓存最近搜索结果

## 九、补充功能模块（用户可能未考虑到的）

### 9.1 数据备份与恢复模块

**重要性**：⭐⭐⭐⭐⭐（核心功能）

党政机关材料非常重要，数据丢失后果严重，必须有完善的备份机制。

**功能设计**：
- **自动备份**：
  - 定时自动备份（每日/每周可选）
  - 保留最近N个备份文件
  - 备份内容包括：数据库文件、配置文件、自定义词典
- **手动备份**：
  - 一键备份按钮
  - 选择备份位置（本地/移动硬盘/网络路径）
  - 备份文件命名：`backup_YYYYMMDD_HHMMSS.db`
- **数据恢复**：
  - 从备份文件恢复
  - 恢复前预览备份数据概要
  - 支持选择性恢复（仅恢复部分数据）
- **备份提醒**：
  - 长时间未备份时提醒用户

**数据库表**：
```sql
CREATE TABLE backup_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    backup_path TEXT NOT NULL,
    backup_size INTEGER,
    backup_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    backup_type TEXT DEFAULT 'manual',
    note TEXT
);
```
